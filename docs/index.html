Drone cheat sheet

Skip to end of metadata
  • Created by Sebastien GERVAIS, last modified on Mar 12, 2018
Go to start of metadata

This page provides some tips and tricks for drone.io

Information

We have 2 drone instance:

  • Drone cloud (also named "fpfis-drone-external"): https://drone.fpfis.eu/
  • Drone internal: https://ec.deploy.fpfis.eu. Used for deployment.

Link

  • Manually triggering failed drone deployments

External:

  • http://docs.drone.io/
  • Drone cli installation: http://docs.drone.io/cli-installation/

Drone Deploy (internal)

Server: ec2-user@ec.deploy.fpfis.eu

  • config: ~/docker-compose.yml
  • update container : docker-compose up -d

Agent : root@vs-fpfis-ci

  • config : /srv/drone/docker-compose.yaml
  • update container : docker-compose up -d

Reboot proxy container: 

$ docker run --detach --net=host --privileged -e http_proxy="http://127.0.0.1:3128" -e https_proxy="http://127.0.0.1:3128" fpfis/forgetproxy

Drone Cloud (external)

Server: ec2-user@drone.fpfis.eu

  • ~/docker-compose.yml <=> docker compose file
  • SSL certificate: /srv/drone/server/

NE-Pipelines fallback: ec2-user@drone.fpfis.eu:/var/www/html/

DB connect:

$ ssh root@drone.ne-dev.eu
$ mysql -h %dbserver% -u %dbpassword% -D drone_prd_int_db -p

drone cli local

Run test:

$ drone exec --build-event push --commit-branch release/drone

drone cli server

Connect
# Load proxy settings
$ . ~/.proxyrc
# Export drone server credentials
$ export DRONE_SERVER=https://drone.fpfis.eu/
$ export DRONE_TOKEN=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# drone cli
$ drone --help
$ drone repo ls

Token can be found on drone UI: https://drone.fpfis.eu/account/token

Update existing secret
drone secret update "ec-europa/c4dweb-reference" --name="deploy_key_test" --event=deployment
Add a SSH private key as secret
# Add new secret
$ drone secret add --repository ec-europa/ami-reference --name=deploy_key_acceptance --value @/home/user/fpfis/.ssh/myprivatekey
# Update existing secret
$ drone secret update --repository ec-europa/ami-reference --name=deploy_key_acceptance --value @/home/user/fpfis/.ssh/myprivatekey
Stop build
$ drone build stop ec-europa/europa-component-library 115

Tips and tricks

Stop build:

When build is cancelled but still running we need to update data on sqlite db

2 parameters:

  • repo_name
  • build_number

$ UPDATE builds SET build_status = 'failure', build_finished = strftime('%s','now') WHERE build_number=28;

We cannot use drone cli because the build is considerate as 'non-running':

$ drone build stop ec-europa/ami-reference 28
client error 400: Cannot cancel a non-running build

Delete build

DELETE b
FROM builds b
  INNER JOIN repos ON b.build_repo_id=repos.repo_id 
WHERE repos.repo_name like ('%c4dweb%') 
  AND repos.repo_owner = "ec-europa" 
  AND b.build_number=175

For internal, please disable push & pull requests

update repos set repo_allow_pr=0,repo_allow_push=0;

Add a forked repo outside the org

It's usually not recommended to do so, the best is to work with pull requests on the reference repo to keep control of the pipeline.

If it is needed anyway, the repo can be enabled through the CLI :

drone repo add username/repo-name

After this, it is also necessary to ensure the repo uses a custom .drone.yml path :

drone repo update username/repo-name --config=.drone-dev.yml

This will ensure the reference repository .drone.yml stays out of the repo and in ne-pipelines.
